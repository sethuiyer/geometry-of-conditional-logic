    Topology-driven exact repair for structured discrete systems

    A three-layer architecture for constraint problems where you commit some variables and repair the rest without recomputing from scratch — using Chinese Remainder Theorem residues as an exact state-update primitive.

    CRT Arithmetic Exact Repair Lock-and-Restore Specialized Primitive Not a Universal Solver

    The Problem with If/Else

    Standard neural networks struggle with if/else logic because the Heaviside step function has zero gradients everywhere except at the boundary — where the gradient is infinite. Gradient descent is a blindfolded hiker standing on a flat plane, unable to feel which direction leads downhill.

    Heaviside step function
    $$H(x) = \begin{cases} 0 & \text{if } x < 0 \\ 1 & \text{if } x \ge 0 \end{cases}$$

    Its derivative is the Dirac delta — zero everywhere, infinite at zero. A neural network on the flat part has no signal at all.

    The industry works around this with sigmoid gates, Gumbel-Softmax distributions, and straight-through estimators. But as the research shows, these are "foam padding over a jagged staircase" — fake smoothness imposed from the outside rather than discovered.

    What if conditional logic has an inherent continuous geometry — and we just haven't learned to see it yet?

    The Core Insight

    Instead of fighting discrete logic with continuous approximations, this project flips the question: What if you encode Boolean states as residues of prime moduli and let a single integer hold the entire truth table?

    Chinese Remainder Theorem

    Encoding logic in prime residues

    Assign each Boolean condition a distinct prime $p_i$. For pairwise coprime primes, there exists a unique Master Coordinate $X$ satisfying all residues simultaneously:

    $$X \equiv a_1 \pmod{p_1}, \quad X \equiv a_2 \pmod{p_2}, \quad X \equiv a_3 \pmod{p_3}$$

    A single integer encodes an entire truth table. Example: target [1, 0, 1] with primes [2, 3, 5] → X = 21, because 21 mod 2 = 1, 21 mod 3 = 0, 21 mod 5 = 1.

    This bijection between integers and residue vectors is the foundation of everything that follows. One number, one manifold, many logical states.

    Differentiable Surrogate

    The cosine wave loss

    Taking a modulo gives a sawtooth wave — discontinuous and gradient-killing. The fix: replace it with a smooth cosine wave whose minima align exactly with the target residues:

    Smooth residue loss
    $$L_i(z) = 1 - \cos\!\left(\frac{2\pi}{p_i}(z - a_i)\right)$$ $$L(z) = \sum_{i=1}^n L_i(z)$$

    The gradient is never zero unless $z$ sits exactly at the valley bottom. Multiple prime waves superimpose into a complex landscape of ridges and basins.

    Each prime creates its own frequency. Superimpose them and you get moiré interference patterns — chaotic terrain where gradient descent gets permanently trapped. Two out of three constraints satisfied looks like a local minimum. That's the "Moiré Trap."
    The Escape Mechanism · 1958

    Garner's Algorithm as a "Teleporter"

    When gradient descent gets stuck, you need a way to jump out of local minima without disturbing already-solved constraints. The answer: Garner's Algorithm, a 1958 method for reconstructing a number from its CRT residues, turns out to be exactly the right tool.

    Garner's mixed-radix reconstruction
    $$X = v_1 + v_2\,p_1 + v_3\,p_1p_2 + \cdots$$

    Each coefficient $v_i$ is computed via modular inverses. The calculation is fully differentiable — it's just multiplication and addition. This is the "safe corridor": an exact algebraic path from any starting point to the target coordinate, preserving every already-solved residue.

    The most important architectural decision: z (the CRT coordinate) is not the operational state. It is an invariant certificate — reconstructed lazily to prove consistency. The solver mutates residues and locks; z is a checkpoint, not a steering wheel.
    Core Primitive

    The Incremental CRT Jump

    The fundamental operation is an exact state transition that changes one variable's value while preserving all locked residues:

    The safe-corridor jump
    $$z' = z + kM, \qquad M = \prod_{p \in S} p$$ $$k \equiv (c - z) \cdot M^{-1} \pmod{p_t}$$

    Set variable t to value c. Because M is divisible by every locked modulus, z mod p is unchanged for all locked primes. This is an exact arithmetic operation — no approximation, no tearing down prior commitments.

    Why this matters: In shift scheduling, if nurse A is already assigned to Monday and nurse B to Tuesday, you can reassign nurse C to Wednesday via one CRT jump without re-optimizing the entire roster. The locked residues stay locked.
    garners_jump Exact residue-preserving transition · Core of all search solvers 
    def garners_jump(current_z, target_val, prime_idx, solved_primes):
    p_target = PRIMES[prime_idx]
    M = 1
    for p in solved_primes:
        M *= p
    diff = (target_val - current_z) % p_target
    k = (diff * mod_inverse(M, p_target)) % p_target
    return k * M
    Used in prime_n_queens.py, prime_timetable.py, and all constraint-solving demos.

    What This Is (and Isn't)

    It IS: An exact lock-and-repair primitive for structured discrete assignment problems. A mechanism for preserving committed decisions while locally fixing disruptions. A bridge between continuous optimization and combinatorial constraint satisfaction.
    It is NOT: A universal optimizer. A P=NP claim. A replacement for CP-SAT on general problems. A production-ready product — it's a research codebase with known scaling limitations.
    Congruence invariance in discrete arithmetic is exactly equivalent to monodromy invariance in continuous topology. The frustrations in gradient descent are not random noise — they are literal physical topographies of constraint violations.
