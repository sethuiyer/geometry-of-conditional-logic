Working Notes

Residue-preserving local state transitions over overlapping constraint groups.

This repository started from the idea of encoding discrete assignments as modular residues and repairing solved commitments with exact CRT jumps. It has since matured into a cleaner distillation: distributed exact commitment systems over overlapping local manifolds (rows, columns, boxes, lanes, etc.), where each local jump preserves all already-solved residues exactly.

The result is a layered research notebook more than a finished theory. There is a real mathematical core here, a set of useful exact-search demos, some exploratory framing around overlapping modular charts, and now a benchmark record showing where the approach helps and where standard tools still remain stronger.

Why care? Because a lot of practical systems sit in the uncomfortable space between continuous learning and discrete commitments. Schedulers, dispatchers, allocators, safety controllers, and routing systems often need to make exact symbolic choices, keep some of them fixed, and repair the rest without tearing everything down. This repo is an attempt to find a compact mathematical language for that setting: one where discrete state stays interpretable, updates can preserve what already works, and search can still borrow some structure from analysis rather than starting from raw brute force every time. The most useful mental model is no longer "one scalar holds the state," but distributed exact commitment systems over overlapping local manifolds — exactly the setting where schedulers, allocators, and safety controllers live.

The sturdy part

The cleanest part of the project is the arithmetic primitive: encode choices as residues inside local constraint groups (rows, columns, boxes, lanes, etc.), then use exact incremental CRT jumps to set new values while preserving every already-solved residue in that group exactly. No global modulus explosion. No tearing down solved commitments.

The useful framing

The most productive way to think about the project is as a specialized search-and-repair idea for structured discrete problems. It is much more convincing in that role than as a sweeping claim about optimization in general.

Specialized primitive Not a universal solver
Executive summary: the repository is easiest to appreciate if you treat it as an experiment in using modular arithmetic structure as an exact state-update primitive for constrained discrete systems. The most interesting practical idea here is not "prime topology replaces optimization," but "CRT-preserving updates may be useful for commitment-preserving repair."
The true core abstraction: residue-preserving local state transitions. Encode discrete state as residues of one or more CRT coordinates. Update a variable by jumping its residue exactly, with no approximation, while preserving all other committed residues. That is the mechanism — everything else (cosine losses, manifold language, SAT landscapes) is either a smooth objective built on top of it or a metaphorical framing stretched over it.
Math Core

The exact mechanism underneath the strongest demos.

Most of the repository reduces to one scalar coordinate $z$, a set of pairwise coprime moduli, and either a differentiable residue surrogate or an exact incremental CRT update.

Residue encoding

$$r_i(z) = z \bmod p_i$$ $$z \pmod P \longleftrightarrow (r_1,\dots,r_n), \qquad P=\prod_i p_i$$

When the moduli are pairwise coprime, one scalar coordinate encodes one full discrete state vector.

Cosine residue loss

$$L_i(z)=1-\cos\left(\frac{2\pi}{p_i}(z-a_i)\right)$$ $$L(z)=\sum_i L_i(z)$$ $$\frac{dL}{dz}=\sum_i \frac{2\pi}{p_i}\sin\left(\frac{2\pi}{p_i}(z-a_i)\right)$$

This is better thought of as an exact smooth objective for congruence satisfaction: its global minima occur exactly at the target residue classes. The heuristic part begins when you ask gradient descent to find those minima reliably.

Garner / CRT reconstruction

$$z = v_1 + v_2 p_1 + v_3 p_1 p_2 + \cdots$$

Several files compute an exact target coordinate from a desired residue pattern. That is the arithmetic backbone of the “teleporter” story.

Incremental safe jump

$$z' = z + kM, \qquad M=\prod_{p \in S} p$$ $$k \equiv (c-z)M^{-1} \pmod{p_t}$$

This is the strongest real mechanism in the repo. It sets a new residue while preserving all already-solved ones exactly. The best exact-search demos use this directly.

Exact

CRT decoding, Garner reconstruction, and residue-preserving jumps are exact arithmetic operations. They do exactly what they say they do, with no approximation layer in the update itself.

Exact lifts

The cosine objectives and some of the energy constructions are not arbitrary hacks. They are smooth lifts with the right zero sets or the right violation geometry for the discrete structure they are trying to represent.

Where heuristics enter

Heuristics mostly enter through optimization schedules, thresholding, ghost-residue semantics, and search budgets. In other words, the weak point is usually the solver strategy or extension regime, not the algebraic construction itself.

def garners_jump(current_z, target_val, prime_idx, solved_primes):
    p_target = PRIMES[prime_idx]
    M = 1
    for p in solved_primes:
        M *= p
    diff = (target_val - current_z) % p_target
    k = (diff * mod_inverse(M, p_target)) % p_target
    return k * M
Benchmarks

What the benchmarks taught us.

Two benchmark scripts now anchor the repo: an internal scaling harness and a direct comparison against OR-Tools CP-SAT on the same generated instances. They are less about “winning” and more about understanding where this style of search has leverage.

Reading note: the CRT search looks good on the planted-feasible timetabling instances tested here, and it remains interesting as a repair primitive. On the inventory benchmark, though, CP-SAT becomes the stronger general-purpose optimizer once size and objective quality matter.
